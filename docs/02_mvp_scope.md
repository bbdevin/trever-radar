# 02 版本範圍與驗收標準

## V1 — 盤後籌碼雷達(目標:做得出來、每天真的會用)

### 包含

| 類別 | 項目 | 備註 |
|---|---|---|
| 帳號 | 登入(Laravel Breeze)、admin 建帳號、2 角色(admin/user) | 無自助註冊 |
| 資料 | 股票主檔、日K(含還原價)、權證主檔+每日成交、分點買賣超(FinMind)、法人買賣超、融資券、注意/處置股 | 歷史回補 ≥ 1 年(分點視 FinMind 額度,至少 6 個月) |
| 計算 | 技術指標(MA5/10/20/60、多頭排列、20日新高、量比、RSI14、KD、MACD)| 純 PHP,golden-file 測試 |
| 評分 | 權證異動分、分點籌碼分、技術分、題材分、法人分、風險扣分、盤後綜合分 | 規則見 04 |
| 產出 | 每日觀察清單(Top 30,含理由文字、風險提醒、觀察價/失效價)、隔日監控池(給 V2 用,先產出存著) | 18:30 前完成 |
| 個股頁 | K 線圖(lightweight-charts)+ 均線 + 量、每日前 15 大買/賣超分點、單一分點在該股的歷史買賣列表與 K 線標記、分點連買天數 | 分點績效統計只做「買後 N 日報酬」簡表 |
| 分點 | 連買榜、集中度榜、地緣券商榜(人工對照表)、關鍵分點名單(人工維護)、隔日沖分點名單(人工+規則) | |
| 題材 | 人工維護題材與成分股、簡易題材熱度(成分股漲幅/量能統計) | |
| 系統 | 自選股、匯入紀錄與資料健檢告警、資料源狀態頁 | |

### 明確不做(V1)

WebSocket、任何盤中功能、Fugle 串接、自動下單、元大 API、AI/LLM 功能、金流、多租戶、公開註冊、完整回測引擎、分點可信度完整模型、權證分點明細、W底/頸線型態辨識、手機 App(用 RWD 網頁)。

### V1 驗收標準

1. 交易日 18:30 前,系統無人工介入自動產出觀察清單;連續 10 個交易日成功率 ≥ 9/10。
2. 任一入選股票,點開能看到:各分項分數、至少 3 條具體理由(含數字)、至少 1 條風險提醒、觀察價與失效價。
3. 個股頁能在 2 秒內載入 240 日 K 線 + 分點標記;點任一分點能列出其近 1 年在該股的每日買賣超與收盤價。
4. 任一資料源匯入失敗或筆數異常(±30% vs 20 日均值),30 分鐘內收到告警通知。
5. 除權息日,還原價與指標無跳空失真(抽 3 檔人工核對)。
6. 評分模組有 golden-file 測試:固定輸入資料集 → 固定分數輸出,CI 全綠。

## V2 — 盤中發動雷達 + 可信度 + 簡易回測

### 包含

1. Fugle MarketData 串接(免費版起步,監控池 ≤ 免費額度;不足則升級開發者方案或用 REST snapshot 輪詢補)。
2. Node.js 盤中 worker:訂閱監控池逐筆/1分K,計算大單、主動買比、量比、VWAP、突破;事件寫 DB + Redis pub。
3. 盤中訊號 A/B/C 分級 + 理由文字 + 追高/等回測判定。
4. 盤中雷達頁(Reverb WebSocket 推播)+ Telegram 或 LINE Notify 替代方案通知(LINE Notify 已於 2025 年停服,用 Telegram Bot,免費)。
5. 分點可信度分數(規則見 04 §5):勝率、買後報酬、隔日沖偵測、近效性。
6. 訊號績效追蹤:每日批次回填觀察清單與盤中訊號的 1/3/5/10/20 日後續報酬;訊號歷史頁 + 各規則勝率統計。
7. 個股籌碼 K 線疊加盤中訊號標記、布林通道、ATR。

### V2 驗收標準

1. 盤中訊號從成交發生到前端顯示 ≤ 5 秒(監控池 50 檔內)。
2. 監控池 50 檔全日運行,Fugle 額度不超限、worker 斷線自動重連並補快照。
3. 每個 A 級訊號都有:觸發時間、理由(含數字)、當下價、風險判定;盤後全部落地可查。
4. 訊號歷史頁可查任一日清單/訊號的後續報酬與分規則勝率。

## V3 — 深度回測 + 題材智慧 + 個人券商輔助(先驗證 V2 有價值再啟動)

1. FinMind 歷史資料補深(3–5 年),參數化回測:調整評分權重 → 重算歷史清單 → 比較績效。
2. 國外題材連動:美/日/韓概念股對照表(人工起步)+ 美股夜盤漲跌 → 台股供應鏈提示。
3. 新聞熱度(FinMind news / 自建 RSS)+ LLM 題材摘要(每日一次批次,控 token)。
4. 分點進階模型:分點×題材勝率、分點聯動偵測(多分點同步)、僅特定股有效偵測。
5. 元大 SPARK API:僅個人帳號、僅查詢(庫存/損益)與手動確認下單頁;獨立服務、獨立憑證保管,絕不自動下單,不給其他使用者。

### 你猜測的修正

- V1、V2 猜測正確。
- V3 的「回測」拆一半到 V2(訊號績效追蹤),V3 只做「參數化重跑歷史」的重回測。
- AI 功能全部押後到 V3,且僅每日批次摘要,不做即時 AI。
