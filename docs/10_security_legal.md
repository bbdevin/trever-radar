# 10 資安、權限與法規

## 1. 帳號與權限

- 登入:email + 密碼(Laravel Breeze)+ 強制 2FA(TOTP,10 人系統也值得,因為含金融決策資訊)。
- 無自助註冊、無邀請碼系統:admin 後台直接建帳號(10 人規模,邀請碼是過度工程)。
- 角色兩種:`admin`(你:使用者管理、參數設定、資料源設定)、`user`(看)。V1 不做細粒度權限。
- Session:7 天、單裝置數不限;敏感頁(系統設定)re-auth。
- 稽核:admin 的參數改動、使用者操作寫 `audit_logs`。

## 2. 金鑰與憑證

- FinMind/Fugle/Telegram token:一律 `.env`,不進 git;`.env.example` 只放鍵名。VM 上檔案權限 600。
- 10 人私用不需要 Vault/KMS(過度工程);但 repo 要裝 gitleaks pre-commit 防手滑。
- 資料庫備份檔含市場資料無個資敏感,但仍加密上傳(rclone crypt)。

## 3. 元大 SPARK API(V3,特別警戒)

1. 憑證(CA)與 API key 只存在**獨立的 PersonalBroker 容器**,與主系統網路隔離,只有你的 admin 帳號 + 二次 2FA 可觸達其頁面。
2. 其他 9 位使用者在 UI 上完全看不到此功能存在。
3. 功能白名單:查庫存、損益、帳務;下單頁 = 手動填單 + 確認彈窗 + 每日筆數/金額上限,**無任何自動觸發下單的程式路徑**(訊號系統與下單模組零 import 關係,物理隔離)。
4. 券商 API 行情不得轉供系統其他使用者(條款風險)。

## 4. 法規(台灣,務實判讀;重大變動前請諮詢律師)

1. **投顧法紅線**:對不特定人提供個股分析建議並**收取報酬** → 需投顧執照。本系統:特定少數人(≤10 熟識)、免費、僅資料統計與觀察訊號、附免責 → 風險低。守住三件事:**不收費、不公開、不寫「建議買進」**。
2. 訊號用語規範:全系統文案禁用「買進/賣出/目標價/建議」,只用「觀察/觸發/風險提醒/訊號失效」。這不只是法務,也是產品誠實性。
3. **交易所資料授權**:TWSE/TPEx 資料再散布或商用需簽資訊使用契約。私人 ≤10 人低風險;**不要**把網站公開、不要開放註冊、不要截圖大量轉發。FinMind/Fugle 依其條款使用,資料不轉售。
4. 免責聲明(登入強制同意 + 每頁 footer):「本系統僅彙整公開市場資料供個人研究,非投資建議;訊號不保證獲利;投資人應自行判斷並承擔風險。」
5. **若未來想開放更多人**:那是不同的產品——需要交易所資訊契約、投顧合規檢視、個資法(PDPA)告知義務、資安義務。屆時重新立項,不要從這套私用系統「順便」長出去。
6. 個資:只存 email 與使用紀錄,不存任何券商帳號資料(元大憑證除外,見 §3),被駭最大損失 = 觀察清單外洩,可接受。
