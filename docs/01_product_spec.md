# 01 產品規格 — Trever Radar

副標:盤後找籌碼,盤中看發動。

## 1. 重新定義

Trever Radar 不是看盤軟體,也不是選股網站的複製品。它是一個**個人化的籌碼異常偵測管線**:

> 每天收盤後,系統把「權證異動、分點連買、地緣券商、關鍵分點、法人動向、技術轉強、題材熱度」壓縮成一份 20~30 檔、每檔都附觸發理由與風險提醒的觀察清單;隔天盤中,只對這份清單 + 自選股做大單與量價轉強偵測,發出分級訊號。

關鍵差異化(相對於現成工具如籌碼K線、XQ、神秘金字塔):

1. **跨訊號交集**:市面工具各自給單一面向(權證、分點、技術),本系統的價值在「昨日籌碼好 ∩ 今日盤中發動」的交集,以及可自訂的權重。
2. **可解釋**:每檔股票輸出人話理由,不是黑箱分數。
3. **可回測**:所有訊號落地保存,可事後驗證哪些規則有效,持續汰弱留強。
4. **私有、可控、便宜**:不受制於第三方網站改版或收費牆(分點資料除外,明確付費)。

同時誠實面對限制:分點資料是 T+1 盤後資料;盤中看不到分點;權證的「主力提前卡位」只能用成交異常間接推論,無法直接證明。

## 2. 產品定位

- 類型:私人投研工具(internal tool),非 SaaS、非投顧服務。
- 使用者:你本人 + 熟識朋友,≤10 人,邀請制。
- 法律定位:僅提供市場公開資料之整理與統計觀察,附風險提醒,不構成投資建議,不收費,不保證獲利,不下單。(詳見 10_security_legal.md)

## 3. 使用者與核心場景

### 人物誌

- **P1 主要使用者(你)**:懂籌碼概念,每天有 30~60 分鐘看盤後資料,盤中可瞄手機。要的是「幫我把該看的濃縮好」。
- **P2 朋友**:程度不一,只看觀察清單和訊號通知,不會調參數。

### 核心場景(依優先序)

| # | 場景 | 時間 | 版本 |
|---|---|---|---|
| S1 | 晚上打開 App,看今日觀察清單,每檔看理由與風險,挑 3~5 檔進自選 | 18:30 後 | V1 |
| S2 | 點進個股,看籌碼 K 線:哪些分點在買、買多久、這分點過去準不準 | 盤後 | V1 |
| S3 | 盤中收到通知:「XX 出現 A 級發動訊號」,打開看理由,自行決定 | 09:00–13:30 | V2 |
| S4 | 週末回顧:上週訊號後續表現如何,哪類規則勝率高 | 週末 | V2/V3 |
| S5 | 看題材輪動:哪個題材熱度升溫,成分股籌碼誰最好 | 盤後 | V1 簡易/V3 完整 |

## 4. 產品架構(功能地圖)

```
Trever Radar
├─ 盤後雷達(V1 核心)
│  ├─ 每日觀察清單(綜合分數 + 理由 + 風險 + 觀察價)
│  ├─ 權證異動榜
│  ├─ 分點追蹤(連買榜/集中度榜/地緣/關鍵分點)
│  └─ 題材面板(人工標籤 + 簡易熱度)
├─ 個股籌碼頁(V1 基礎 → V2/V3 強化)
│  ├─ K 線 + 均線 + 量 + 訊號標記
│  ├─ 每日前 15 大買賣超分點
│  ├─ 分點疊加 K 線(足跡)
│  └─ 分點歷史績效與可信度(V2)
├─ 盤中雷達(V2)
│  ├─ 監控池(昨日清單 + 自選)
│  ├─ 大單/量價/VWAP/突破訊號,A/B/C 分級
│  └─ 通知(Web push / Telegram/LINE)
├─ 回測與訊號歷史(V2 簡易 → V3 完整)
├─ 自選股(V1)
└─ 系統(資料源狀態、匯入紀錄、使用者管理)(V1)
```

## 5. 版本路線總覽(細節見 02_mvp_scope.md)

- **V1(約 6–8 週兼職開發)**:盤後籌碼雷達 + 每日觀察清單 + 基礎個股籌碼 K 線。全部 T+1 資料,零盤中依賴。→ 你的猜測正確。
- **V2(+6–8 週)**:盤中大單與技術轉強雷達(僅監控池)、通知、分點可信度模型、簡易訊號回測。→ 你的猜測正確。
- **V3(視 V1/V2 實戰價值再決定)**:FinMind 深度歷史回測、國外題材連動、新聞/AI 題材摘要、進階分點模型、元大 SPARK 個人輔助(查詢類,非下單)。→ 你的猜測大致正確,但建議把「回測」提前一部分到 V2(不回測就無法汰弱訊號規則,V3 才做太晚)。

## 6. 主動補充:你沒提但重要的事

1. **訊號績效自我追蹤是第一級公民**:每天的觀察清單與盤中訊號都要記錄「後續 1/3/5/10/20 日報酬」,否則三個月後你不知道該信哪個分數。這比多做任何一個新訊號都重要,已排入 V2 初。
2. **資料品質告警**:免費資料源常改版。每日匯入後要跑健檢(筆數、缺漏股票數、與前日差異),異常就發通知,否則你會在「資料早就斷了」的清單上做決策。V1 就要有。
3. **除權息與股本變動**:分點歷史績效與技術指標要用還原價,否則除息日全部訊號失真。V1 就要處理(FinMind 有還原價,或自行用除權息表調整)。
4. **隔日沖分點是雙面刃**:它們的買超是最常見的假「籌碼好」來源。V1 就要維護一份已知隔日沖分點名單(人工 + 規則偵測),在分點分數中扣分,而不是等 V2 模型。
5. **觀察清單要有「為什麼沒入選」**:自選股即使沒入選,也顯示它今天各分項分數,避免黑箱感。
6. **停損思維內建**:觀察價除了「突破觀察價」,也給「跌破失效價」(例如跌破 5 日線或箱型下緣即訊號失效),風險提醒才完整。
