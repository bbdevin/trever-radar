## 2. Part A:盤中訊號雷達(Intraday Radar)

**一句話**:盤後選好「該盯誰」(Armed 池 + 自選),盤中本機 worker 盯這一小池,抓大單/爆量/急拉/發動,經 Supabase 中繼到前端盤中面板。

### 2.1 架構(取代 docs/08 §2 舊設計)

```
盤後(既有管線)                     盤中(新增,僅使用者本機)
export: Armed 池 + watch/stop 價     worker(Python,08:55–13:35)
      + 每檔「時刻量能基準」曲線  →   Fugle MarketData(使用者免費 key)
                                      ↓ 規則判定 I-1~I-4
                                    Supabase intraday_signals(+heartbeat)
                                      ↓ supabase-js 輪詢 30s(或 Realtime)
                                    前端首頁「盤中」面板(僅盤中時段顯示)
```

- **監控池上限 ~30 檔**:今日 Armed(通常 0–10)+ 自選(≤20)+ 可選綜合榜前 N。WS 逐筆給前 5–10 檔,其餘 REST 輪詢 30–60s——受 Fugle 免費額度約束,**不做全市場盤中掃描**。
- **中繼用 Supabase**(已有 client/Auth/RLS 基礎):worker 以 service key(僅存使用者本機 .env)寫入;前端 authenticated 讀。不新增任何常駐伺服器。
- **誠實限制(UI 必標)**:worker 沒開機=沒有盤中資料,面板顯示 heartbeat 離線狀態;訊號為觀察提醒非建議(docs/10 語氣)。

### 2.2 訊號規則(規則模板,不用 LLM;門檻為 V1 起始值,Shadow 一週後校準)

| 代碼 | 規則 | 資料需求 |
|---|---|---|
| I-1 大單 | 單筆成交金額 ≥ 分級門檻(依股本/日均額分級,如 ≥500萬/1000萬)或 1 分鐘內外盤淨差 ≥ 門檻 | Fugle 逐筆(WS 池) |
| I-2 爆量 | 當日累積量 ÷ 同時刻 ADV20 基準 ≥ 2.0 | 盤後預算每檔時刻量能基準曲線隨 JSON 下發 |
| I-3 急拉 | 5 分鐘漲幅 ≥ 2% 且該窗量增 | 1 分 K(WS 聚合或 REST) |
| I-4 發動 | **Armed 池**股價突破 `watch_price` 且 I-2/I-3 任一成立 → 即 docs/22 的盤中 Triggered | Armed 名單 + watch_price(既有) |

### 2.3 階段與依賴

| 階段 | 內容 | 估時 | 依賴 |
|---|---|---|---|
| I0 PoC | 本機腳本訂 2–3 檔,驗證 Fugle 免費額度/欄位粒度(逐筆是否含內外盤/單量),產可行性報告與門檻建議 | 半天 | **使用者申請 Fugle API key** |
| I1 資料層 | Supabase `intraday_signals`/`worker_heartbeat` 表 + RLS(SQL 檔照 watchlist 慣例);export 輸出 Armed 池與時刻量能基準 | 1 天 | docs/22 Armed A1(名單)先行 |

**風險**:Fugle 免費方案額度/條款變動(I0 先驗);訊號雜訊(先 Shadow,不進任何分數);盤中面板誘發追價(風險提示常駐)。**不做**:自動下單、雲端常駐 worker、全市場掃描。

## 3. Part B:分點追蹤視角(常見分點 → 近 N 日買最多)

**一句話**:在 /branch 把排行榜與分點視角合併——點任一「常見分點」(tracked_branches 手動+自動),看它近 1 / 5 / 10 / 20 / 自訂 N 日淨買超最多的股票。

### 3.1 設計

- **入口合併**(使用者同意合併):排行榜 tab 的分點列點入 → 展開/切換至該分點視角;另設分點下拉快速切換。不新開一級路由。
- **視角內容**:期間選擇 pills(1/5/10/20/自訂 N,自訂上限=資料深度);表格(沿用 V2 語意化 table 慣例):股票|淨買超張|估算金額(張×1000×期末收盤)|平均佔成交比|期間漲跌%;Top 15,依淨買張排序;點股票入個股頁。
- **資料層**:export 為每個 tracked branch 產 `branches/track/{branch_key}.json`——近 120 日「每日×每股淨買超」精簡列表(`[date, stock_id, net_lots, pct]` + 期末收盤對照),**前端在客戶端做任意區間加總**(含自訂日),免為每種天數各出一檔。體積估:每分點 50–200KB×追蹤 10–30 個,可控;股名由既有搜尋索引補。
- **依賴與時機**:VPS 分點歷史(490 日)回灌後資料才有深度——**回灌完成即可開工,時機剛好**。自訂日上限先 120,資料到位後可放 240。
- **誠實限制照舊標注**:資料僅每日前 15 大買/賣超,冷門單不可見;分點≠單一人。

### 3.2 階段

| 階段 | 內容 | 估時 |
|---|---|---|
| B1 export | `branches/track/*.json` 聚合輸出 + 單元測試(種子 DB) | 0.5 天 |
| B2 前端 | /branch 排行榜整合分點視角 + 期間 pills + 表格 | 1 天 |
