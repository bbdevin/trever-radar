# 07 前端頁面與 UI/UX

## 0. 資訊架構(重整過,非 15 個平行頁面)

你列了 15 頁;10 人私用系統做 15 個一級頁面會稀釋開發量也難導航。重整為 **5 個一級區 + 個股頁作為樞紐**:

```
① 今日雷達(首頁)     — 觀察清單 + 大盤風險條 + 資料狀態燈
② 盤中雷達(V2)       — 即時訊號流 + 監控池狀態
③ 探索                 — Tab:權證異動｜分點連買｜集中度｜地緣｜關鍵分點｜分點績效榜｜題材
④ 個股頁(樞紐)       — Tab:總覽｜籌碼K線(足跡)｜分點明細｜權證｜訊號歷史
⑤ 我的                 — 自選股｜訊號歷史/回測(V2)｜系統(資料源狀態、匯入紀錄、使用者管理、設定)
```

所有列表的股票列一律可點 → 個股頁;所有分點名一律可點 → 分點視角(該分點近期買賣的所有股票)。「頁面少、鑽取深」是核心 UX 原則。

## 1. 今日雷達(首頁,V1)

- **用途**:S1 場景,5 分鐘看完今天該看的。
- **頂部**:日期與資料完整度燈(綠=全齊/黃=部分存疑/紅=缺料,點擊看明細)、大盤風險條(指數 vs 60 日線、今日全域門檻是否提高)。
- **主體**:觀察清單卡片列(Top 30)。卡片欄位:代號+名稱(點入個股)、收盤價、漲跌幅、**綜合分(大字)**、五個分項迷你條(分點/權證/技術/題材/法人)、觸發理由前 2 條(完整句)、風險提醒(紅字)、觀察價/失效價、題材標籤 chips、標記 icon:🔑關鍵分點 ⚡權證異動 📍地緣 ⚠️隔日沖。
- **篩選**:分項分數門檻滑桿、題材多選、排除注意股、只看「權證+分點雙高」。**排序**:綜合分/分點分/權證分/漲幅。
- **附區**:權證異動特別榜(未進主榜者)、「自選股今日體檢」(自選未入選者的分數速覽)。
- **手機**:卡片單欄,理由摺疊(點展開);**桌機**:左清單右預覽(點卡片右側直接載個股摘要,免跳頁)。

## 2. 盤中雷達(V2)

- **用途**:S3 場景,盤中瞄一眼。
- **主體**:訊號流(倒序),每列:時間、代號名稱、**級別徽章(A 紅/B 橙/C 灰)**、現價與漲幅、理由一句(含數字)、追高警示標記、距 VWAP %。點入個股頁盤中視圖。
- **上方**:監控池狀態列(N 檔監控中、今日 A×3 B×7、worker 心跳燈)。
- **篩選**:只看 A、只看自選、只看非追高。
- **通知設定**:每人自訂——A 級推 Telegram、B 級僅站內。
- **手機優先設計**(此頁 80% 在手機看):大字、單欄、色盲安全的級別形狀差異(A=實心/B=框線)。

## 3. 探索(V1,Tab 集合)

| Tab | 主欄位 | 排序/篩選 | 情境 |
|---|---|---|---|
| 權證異動 | 標的、權證金額、20日倍數、購售比、連續天數、標的漲幅、權證分 | 倍數/連續天數;排除已漲>5% | 找先行卡位 |
| 分點連買 | 股票、分點、連買天數、累計張/金額、佔成交比、該分點可信度 | 天數/佔比 | 找耐心買家 |
| 集中度 | 股票、買方集中度、vs 20日均、前5大分點列表 | 躍升幅度 | 找籌碼收攏 |
| 地緣 | 股票、地緣分點、城市、淨買、連續性 | 淨買金額 | 找知情資金 |
| 關鍵分點 | 分點、今日買超股票列表、各筆金額 | — | 追特定主力 |
| 分點績效榜(V2) | 分點、樣本數、5日勝率、平均報酬、型態(波段/隔日沖)、可信度分 | 可信度;最少樣本數門檻 | 決定誰值得信 |
| 題材 | 題材、熱度分、今日均漲、上漲比、金額倍數、Top 成分股 | 熱度 | 看輪動 |

桌機:表格;手機:每列摺疊卡。

## 4. 個股頁(樞紐,V1 基礎)

### Tab 1 總覽
今日各分項分數環形圖、理由與風險全文、觀察價/失效價、基本資訊(股本、產業、題材、注意/處置狀態)、近 5 日法人與融資券小表。

### Tab 2 籌碼 K 線(核心,詳見 09_broker_footprint.md)
lightweight-charts 日 K + 量 + 均線;下方分點淨買賣堆疊柱;右側分點面板(當日前 15 買/賣);點分點 → K 線疊標記。篩選:關鍵/地緣/金額門檻/連買/高可信度(V2)。

### Tab 3 分點明細
當日前 15 買/賣超表格(張、金額、佔比、連買天數、隔日沖標記、可信度);點分點 → 該分點在本股歷史操作表(日期、買賣超、收盤價、後 1/3/5/10/20 日報酬)+ 統計摘要 + 評語。

### Tab 4 權證
本股權證列表(價內外、天期、價差、今日金額與倍數)、權證金額 20 日趨勢圖、風險註記。

### Tab 5 訊號歷史
本股歷次入選紀錄與盤中訊號、當時理由、後續報酬(誠實展示,包括失敗的)。

手機:Tab 橫滑;K 線橫屏提示。

## 5. 我的(V1)

- **自選股**:表格 = 觀察清單同欄位 + 自訂備註 + 加入日期與其後報酬;一鍵「今日體檢」。
- **訊號歷史/回測(V2)**:日曆選日期 → 當日清單快照與後續表現;規則勝率統計圖(各 reason code 的 5 日勝率橫條圖)。
- **系統(admin)**:資料源狀態(各 dataset 最後成功時間、筆數趨勢迷你圖、健檢結果)、匯入紀錄表、使用者管理(建帳號/停用)、參數設定(門檻與權重,改動記 audit log)。

## 6. 全域 UI/UX 原則

1. **深色為預設**(看盤慣例),亮色可切。漲紅跌綠(台股慣例,別用美股配色)。
2. 每個分數旁必有 ⓘ,點開顯示計算明細(哪幾條規則給了幾分)——「可解釋」落實到 UI。
3. 免責聲明:登入頁 + 每頁 footer 固定一行 + 首次登入強制同意頁。
4. 所有時間標註資料日(「分點資料:7/4」),避免误以為即時。
5. 空狀態要有教育性:「今日無 A 級訊號——寧缺勿濫是設計目標」。
6. 效能:清單頁一次 API 回傳全部(30 檔),無分頁;個股頁四表合一 endpoint;目標 LCP < 2s。
