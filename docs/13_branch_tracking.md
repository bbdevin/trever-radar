# 13 分點排行與追蹤(規劃;2026-07-07 使用者需求)

> 目標:一張「值得追蹤的分點」排行表 + 每日「這些分點在買什麼」動向表,含權證分點視角。
> 本文件把 04 §2–5(分點模型)與 09(足跡)落成**可直接開工的功能規格**。
> **前提:分點資料尚未解鎖**(見 §6)。資料一到位,本規格即可實作,無需再議。

## 1. 功能一句話

盤後自動回答兩個問題:
1. **誰值得追?** —— 分點可信度排行榜(演算法算出高勝率、會賺錢、非隔日沖的分點)
2. **他們今天買了什麼?** —— 追蹤名單分點的每日買超明細(股票 + 權證)

## 2. 追蹤名單(兩個來源,合併去重)

| 分點 | 使用者標記 | 備註 |
|---|---|---|
| 富邦-新店 | 追蹤 | 市場知名波段分點(待數據驗證) |
| 凱基-三多 | 追蹤 | |
| 元大-大天母 | 追蹤 | |
| 永豐金-內湖 | 追蹤 | |
| 凱基-信義 | 追蹤 | 市場常見隔日沖疑慮,演算法會自動判定型態 |

- 表:`tracked_branches(branch_id, source='manual', added_by, note, added_at)`
- 種子名單只是起點;**名聲交給演算法驗證**——上榜與否看數據,不看江湖傳說。
- 管理介面:系統頁 admin 可增刪(V1 先直接改 DB/seed 檔)。

### 2b. 演算法自動入選(每週六重算,04 §5 可信度分數)

入選條件(全部滿足):
1. 近 2 年「進場事件」≥ 10 次(事件 = 單股單日淨買超 ≥ 該股成交值 1%,連續日合併)
2. 可信度分數 ≥ 70(構成:5 日勝率 30% + 平均報酬 25% + 買點分位 15% + 非隔日沖 10% + 近 90 日仍有效 20%)
3. 非隔日沖(買超次日回吐 ≥70% 的比率 < 60%)
4. 近 90 日仍活躍(事件 ≥ 2)

自動移出:連續 60 日可信度 < 50 或不活躍。`source='auto'`,與手動名單並存,UI 標示來源。

## 3. 排行榜演算法(每日/每週計算)

### 3a. 分點×個股統計(`branch_stock_stats`,05 已定義)

每事件記錄:進場日、均價(以次日開盤計,避免 T+1 偷看)、張數、金額、後 1/3/5/10/20 日報酬。
彙總:事件數、勝率(5日>0)、平均/中位報酬、最大回撤、買點分位(買在近 20 日區間低點程度)、隔日沖率、持有型態(短線/波段)。

### 3b. 分點總排行(跨個股彙總)

```
排行分 = 0.30×勝率(40%→0分,70%→滿分)
       + 0.25×平均5日報酬(0%→0,+5%→滿分)
       + 0.15×買點品質(買低分位)
       + 0.10×規模(近90日買超總金額,log 級距──太小的分點統計無意義)
       + 0.20×近效性(近90日 vs 全期報酬衰減)
隔日沖判定成立 → 單獨列「隔日沖分點榜」,不進主榜(它們是反指標/風險訊號來源)
```

### 3c. 權證分點(同一套,兩個差異)

1. **資料面與跨權證彙總**：權證代號理論上為獨立個體，但因一檔熱門股票常同時有多檔權證發行，分點大戶通常會分散買進多檔。故實作上以「標的股」為中心，將單一分點對同一標的下「所有權證」的買賣張數，乘上該日權證收盤價，加總出該分點對該標的的「權證淨買賣金額」。
2. **統計面與時間維度**：為了捕捉大戶在不同週期的佈局，我們將時間維度切分為 **1D / 2D / 5D / 30D**，並針對每個區間設立「淨買賣超 > 500萬台幣」的門檻。達標的分點會被標註並顯示在前端，以便快速定位大戶的集中火網。

## 4. 頁面規格(掛在 探索 → 分點)

### Tab「分點排行」
| 欄 | 內容 |
|---|---|
| 分點 | 券商-分點名(點入 → 分點視角頁) |
| 來源 | 手動/自動 徽章 |
| 可信度 | 0–100,≥70 亮黃 |
| 5日勝率 / 平均報酬 | 樣本數 <10 顯示「樣本不足」 |
| 型態 | 波段 / 短線 / 隔日沖(紅) |
| 近90日 | 事件數 + 買超金額 |
排序:可信度;篩選:最少樣本數、排除隔日沖、只看手動名單。

### Tab「今日動向」(核心使用場景)
追蹤名單分點今日買超明細,一列一筆:
分點|股票(點入個股頁)|買超張/金額|佔成交量%|連買天數|**該分點在該股歷史勝率**|該股今日綜合分
排序:買超金額;重點:「高可信度分點 + 綜合分 ≥55」的交集列 → 加醒目標記,並回饋到 04 §2 B5 給分。

### 分點視角頁(點任一分點)
近 90 日操作時間軸(買賣超 × 股票)、績效統計卡、擅長產業/題材分布、在追蹤個股上的 K 線足跡連結(09)。

### 權證大戶 Tab (原權證分點)
以 Bento Grid 卡片呈現，並提供 `1D / 2D / 5D / 30D` 的快速切換 (Pill Selector)。
卡片重點展示：分點名稱、標的名稱與代號、醒目的淨買賣金額 (大買/大賣標籤)。僅顯示淨買賣超 > 500萬台幣 的主力大戶。
**互動設計**：支援點擊卡片向下展開明細 (Accordion)，顯示構成該大單的所有權證清單 (代號、名稱、認購/售、貢獻金額與佔比)，並依金額自動排序。

## 6. 資料前提與成本(2026-07-07 修訂:使用者選擇免費爬蟲先行)

| 方案 | 內容 | 成本 | 狀態 |
|---|---|---|---|
| **富邦 ebrokerdj 公開頁**(現行) | 每股每日前 15 大買/賣超分點(張、佔比),無登入無驗證碼,可按日查 | 免費 | **已接入**:`import-branch-trades`,每晚抓綜合分前 80 檔,約 4 分鐘;`branch_trades` 表累積中 |
| FinMind 贊助方案 | 全市場全量分點,結構化 | 約 NT$300–600/月 | 升級選項:要「全市場」或富邦頁失效時啟用 |
| TWSE bsr 爬蟲 | CAPTCHA | — | 不做(已定案) |

免費方案的誠實限制:
1. **只有前 15 大**——小額分點看不到(對追主力夠用)。
2. **只涵蓋每日評分池 80 檔**——冷門股沒有分點史;統計偏向熱門股。
3. 非官方頁面,改版/限流風險 → parser 掛掉會進 import_logs 告警;低頻(每晚一輪、3 秒節流)降低風險。
4. 歷史回補淺:從接入日往後累積;`--date` 可逐日補近月,別暴力回補一年。
5. **統計效力需要時間**:可信度分數要 ≥10 次事件,約累積 2–3 個月後排行榜才可信;之前先出「今日動向」表(§4 Tab 2)。

原 FinMind-only 行動點作廢;實作順序改為:資料累積(已開始)→ 今日動向頁 → branch_stock_stats → 排行榜 → 權證分點。

## 7. 誠實限制(UI 要標)

1. 分點資料 T+1,「今日動向」實為「最近一個交易日」。
2. 分點 ≠ 單一人:大分點混雜多帳戶,統計是機率不是身分證。
3. 勝率以「次日開盤進場」計算,與分點實際成本有差。
4. 名單公開的知名分點常被跟單,訊號會衰減——近效性權重 20% 就是為此設計。
