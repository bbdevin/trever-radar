## WP-H1 首頁榜單依題材分組(純前端,零管線改動)

- **資料已齊**:radar.json `stocks[].themes`(股票卡題材標籤既有)+ `themes[]` 資金流榜(vs20 排序)。
- **互動**:榜單工具列加「排序:分數 | **題材**」切換(記 localStorage)。題材模式下:
  - 每個題材一個 **section**:題材名 + 資金流 badge(`vs20 +X%`,熱度色)+ 檔數;section 依「題材當日資金流 vs20」降冪。
  - 組內卡片按原榜排序;**一檔多題材 → 只歸入當日最熱的那個題材**(避免重複卡);無題材 → 末尾「其他」組。
  - Section header sticky(手機捲動時知道在哪組);組可收合(預設全開,前 3 組之後收合)。
- 適用 tab:綜合 / 市場掃描 / 口袋名單(未來);Armed/Triggered 池與策略 tab 不套(語意不同)。
- 驗收:切換即時、375 sticky 正常、一檔只出現一次、空題材日 fallback 原排序。

## WP-H3 卡片走勢圖改「當日分時」(有一個使用者決策點)

**現況**:spark = 近 30 日收盤線,資訊量趨近零。**目標**:當日分時走勢(相對開盤,紅漲綠跌,平盤虛線)。

**資料方案**:
- **A(建議)盤後 Fugle REST 抓當日 1 分 K**:榜單聯集 ~150–200 檔,60 req/min 限速下 ~3–4 分鐘,加進 daily-market/insti/branches 的 export 前步驟;降採樣 ~60 點寫 `stocks[].spark_day`(+開盤價基準)。
  - ✅ **決策已定(2026-07-12):使用者批准 A 案**——Fugle key 進 GitHub Actions secret(`RADAR_FUGLE_TOKEN`,與 FinMind token 同等級慣例;此為「金鑰只進 VPS .env」政策的獲准例外)。**實作時使用者需親自到 repo Settings → Secrets 新增該 secret**(agent 不碰金鑰值);B 案降為缺資料日的 fallback 呈現。
  - 限制:Fugle 分時**只有當日**可查(隔日補不到)→ 非交易日/缺資料日 fallback 顯示近 30 日線並標小字「30日」;歷史不回補。
- **B(零新金鑰 fallback)當日 OHLC 迷你圖**:開盤→高低範圍→收盤的單日 range bar + 漲跌色。零成本零金鑰,但非「走勢」;使用者不批 A 時採用。
- 前端:sparkline 元件支援兩型(分時線/30日線),tooltip 免(卡片層級);維持 CLS=0(容器定高)。
- 驗收:14:10 後當日分時上線;缺資料 fallback 正常;JSON 體積增量回報(60 點 × 200 檔 ≈ 可忽略)。

## WP-H4 個股頁分點資訊統一(去重疊)

**現況重疊**:「分點」tab = BranchFlowSection(與 K 線視圖同一份)+ 分點分卡 + 理由風險 → 使用者感到兩處重複。

**Planner 建議(A 案):移除「分點」tab,單一真相區**——
- K 線視圖的「分點進出」區升級為唯一分點區:標頭併入**分點分徽章**與**理由/風險 pills**(用 WP-H2 家族色);其下才是既有的期間選擇+買賣超列表+勾選疊圖。
- tabs 收為「K線 / 權證」;舊 `#branch` deep link/tab 參數導向 K 線視圖並捲動至分點區。
- 理由:與 V1/IA「一頁到底掃讀」哲學一致;消滅同構區塊出現兩次的怪感;導覽更少層。
- **B 案(保留 tab)**:tab 只放「當日前15大明細表」不再放聚合區——仍有兩個入口,不推薦;僅當 A 案實作中發現 K 線視圖過長(手機)才退回。
- 驗收:功能零遺失(分點分/理由風險/期間聚合/勾選疊圖/誠實限制都在)、舊連結不 404、375 捲動順暢。
